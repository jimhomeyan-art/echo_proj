## Workflow

### 完整分析流程（按顺序执行）

**第1步：确定交易日期**

```python
from datetime import datetime, timedelta
now = datetime.now()
weekday, current_time = now.weekday(), now.strftime("%H:%M")

if weekday == 5: trade_date = (now - timedelta(days=1)).strftime("%Y%m%d")
elif weekday == 6: trade_date = (now - timedelta(days=2)).strftime("%Y%m%d")
elif current_time < "09:30": trade_date = (now - timedelta(days=3 if weekday == 0 else 1)).strftime("%Y%m%d")
else: trade_date = now.strftime("%Y%m%d")  # 09:30后用当天

print(f"当前时间: {now.strftime('%Y-%m-%d %H:%M:%S')}")
print(f"✅ 必须使用的数据日期: {trade_date}")
```

**15:00后必须用当天收盘数据，严禁用昨天数据！**

**第2步：获取K线数据并校验日期（阻断点）**

```python
df = ak.stock_zh_a_hist(symbol=code, period="daily", ...)
latest_date = df['日期'].max()

if latest_date != trade_date:
    print(f"⚠️ 数据日期不匹配！期望{trade_date}，实际{latest_date}")
    # 必须执行备用方案，不能继续
```

**⛔ 阻断规则：如果数据日期 ≠ 预期交易日，必须：**
1. 立即执行备用数据源（见"API失败备用方案"）
2. 拿到正确日期数据后才能继续分析
3. **严禁用旧数据继续分析！**

**第3步：趋势分析**（详见"中期趋势判断"章节）
- 获取近10个交易日日K线
- 判断趋势方向（上升/下降/横盘）
- 分析均线排列
- 计算MACD、KDJ等技术指标
- 识别支撑位和压力位

**第4步：市场热点扫描**（详见"市场热点扫描"章节）
- 使用 `ak.stock_zt_pool_em(date=trade_date)` 获取全市场涨停
- 按行业统计涨停数量
- 识别2-3个最热板块和龙头股

**第5步：板块联动分析**（详见"板块联动分析"章节）
- 确定个股所属板块

**第6步：分时主力行为分析**（详见"分时主力行为分析"章节）
- 获取当日分时走势
- 识别主力吸筹/出货特征
- 对比近3日分时发现规律

**第7步：生成趋势图**（详见"趋势图生成"章节）

**第8步：输出报告**（详见"输出格式"章节）
- 生成MD报告
- 转换为PDF
- 在对话末尾询问用户持仓情况

- API数据只到昨天 → 直接用昨天数据出报告
- 找不到今天数据 → 说"可能是假期"不验证
- 不尝试备用方案 → 直接放弃

### 规则5：数据比较必须精确，严禁混淆

**突破/跌破的判断必须用精确数值比较：**

| 术语 | 定义 | 判断条件 |
|------|------|----------|
| 突破压力位 | 收盘价 > 压力位 | 12.27 > 12.41? ❌ 否 |
| 跌破支撑位 | 收盘价 < 支撑位 | 精确数值比较 |
| 站稳均线 | 收盘价 > 均线 | 精确数值比较 |

**典型错误**：
- ❌ "12.27元突破了12.41元压力位" → 12.27 < 12.41，没突破！
- ❌ "接近压力位"说成"突破压力位"
- ❌ 盘中最高触及压力位 ≠ 收盘突破

**正确表述**：
- ✅ "12.27元距压力位12.41元还差0.14元，尚未突破"
- ✅ "盘中最高12.37元接近但未突破12.41元压力位"

## 中期趋势判断

### 趋势方向判断

| 判断项 | 上升趋势 | 下降趋势 |
|--------|----------|----------|
| 低点 | 逐步抬高 | 逐步降低 |
| 高点 | 逐步抬高 | 逐步降低 |
| 均线 | 多头排列 | 空头排列 |

**趋势优先原则**：
- 趋势未破坏 → 即使今日跌，不轻易看空
- 趋势已破坏 → 即使今日涨，保持谨慎

### 趋势判断检查清单

必须回答以下问题：

1. **趋势方向**
   - 低点是否逐步抬高？
   - 高点是否逐步抬高？
   - 判断：上升/下降/横盘

2. **趋势强度**
   - 近7日累计涨幅
   - 是否有放量突破
   - 判断：强/中/弱

3. **均线排列**
   - 5日 > 10日 > 20日？
   - 判断：多头/空头/缠绕

4. **今日位置**
   - 今日收盘相对均线位置
   - 判断：趋势未破坏/趋势转弱

### 涨停/跌停判断标准

**⚠️ 重要：不能仅凭开盘=收盘就判断为涨停/跌停！**

| 走势类型 | 判断标准 | 常见错误 |
|----------|----------|----------|
| 一字涨停 | 开=高=低=收=涨停价 | ❌ 开=收就是一字板 |
| T字涨停 | 开=高=收=涨停价，低<涨停价 | 开盘涨停后打开又封住 |
| 尾盘涨停 | 收盘=涨停价，盘中有波动 | 尾盘封住涨停 |
| 低开高走 | 开<收，收≠涨停价 | ❌ 收=高就是涨停 |
| 未涨停 | 收盘 < 涨停价 | 即使涨幅9%也不是涨停！ |

**涨停价计算**：
- 主板/创业板：昨收 × 1.10（保留两位小数，四舍五入）
- 科创板/北交所：昨收 × 1.20
- ST股：昨收 × 1.05

```python
limit_up_price = round(yesterday_close * 1.10, 2)  # 主板
is_limit_up = (close_price == limit_up_price)
```

### K线数据获取代码模板

```python
import akshare as ak
from datetime import datetime, timedelta

# 确定期望的数据日期（使用第1步逻辑）
now = datetime.now()
weekday = now.weekday()
current_time = now.strftime("%H:%M")

if weekday == 5:
    expected_date = (now - timedelta(days=1)).strftime("%Y%m%d")
elif weekday == 6:
    expected_date = (now - timedelta(days=2)).strftime("%Y%m%d")
elif current_time < "09:30":
    expected_date = (now - timedelta(days=3 if weekday == 0 else 1)).strftime("%Y%m%d")
elif current_time >= "15:00":
    expected_date = now.strftime("%Y%m%d")
else:
    expected_date = now.strftime("%Y%m%d")

start_date = (now - timedelta(days=30)).strftime("%Y%m%d")

print(f"={'='*60}")
print(f"当前时间: {now.strftime('%Y-%m-%d %H:%M:%S')}")
print(f"期望数据日期: {expected_date}")
print(f"={'='*60}")

# 获取K线数据
stock_code = "000001"  # 替换为实际股票代码
df = ak.stock_zh_a_hist(symbol=stock_code, period="daily",
                         start_date=start_date, end_date=expected_date, adjust="qfq")

# 打印原始数据并校验日期
print(f"\nAPI原始返回 - 最后7条数据:")
print(df.tail(7).to_string())
print(f"={'='*60}")

# 校验数据日期
if len(df) > 0:
    actual_date = str(df.iloc[-1]['日期']).replace('-', '')
    if actual_date != expected_date:
        print(f"❌ 数据日期错误！期望{expected_date}，实际{actual_date}")
        print("⚠️ 需要执行备用数据源！")
    else:
        print(f"✅ 数据日期正确: {actual_date}")

        latest = df.iloc[-1]
        print(f"\n最新交易日数据:")
        print(f"日期: {latest['日期']}")
        print(f"开盘: {latest['开盘']}")
        print(f"最高: {latest['最高']}")
        print(f"最低: {latest['最低']}")
        print(f"收盘: {latest['收盘']}")
        print(f"成交量: {latest['成交量']}")
        print(f"涨跌幅: {latest['涨跌幅']}%")

        # 判断是否涨停
        if len(df) >= 2:
            yesterday_close = df.iloc[-2]['收盘']
            limit_up = round(yesterday_close * 1.10, 2)
            is_limit_up = (latest['收盘'] == limit_up)
            print(f"\n涨停价: {limit_up}元")
            print(f"是否涨停: {'是' if is_limit_up else '否'}")
```

### 涨停判断标准

| 类型 | 判断标准 | 说明 |
|------|----------|------|
| **一字涨停** | 开盘=最高=最低=收盘=涨停价 | 全天封死涨停 |
| **T字涨停** | 开盘=最高=收盘=涨停价，最低<涨停价 | 开盘涨停后打开又封住 |
| **尾盘涨停** | 收盘=涨停价，盘中有波动 | 尾盘封住涨停 |
| **未涨停** | 收盘 < 涨停价 | 即使涨幅9%也不是涨停！ |

**❌ 常见错误**：
- 错误：收盘价=最高价 → 涨停  （这只是收在最高点，不代表涨停）
- 正确：收盘价 = 涨停价 → 才是涨停

### 龙头股识别

- **涨停时间越早，龙头地位越高**
- **封单额越大，龙头越稳固**
- 收盘价=最高价 ≠ 涨停！必须达到涨停价才算涨停

## 板块联动分析

### 板块地位判断标准

| 类型 | 涨停时间 | 封单量 | 市场辨识度 | 连板情况 |
|------|----------|--------|------------|----------|
| 龙头 | 最早（10:30前）| 大 | 高 | 多数有连板 |
| 中军 | 中等（10:30-14:00）| 中 | 中 | 部分有连板 |
| 后排 | 较晚（14:00后）或未涨停 | 小 | 低 | 很少连板 |

### 板块联动四象限分析

| 场景 | 结论 | 操作 |
|------|------|------|
| 个股强+板块强 | 看多 | 持有，可积极参与，注意仓位管理 |
| 个股弱+板块强 | 可能是低吸机会 | 观察企稳，企稳后可低吸 |
| 个股强+板块弱 | 持续性存疑 | 快进快出，不宜追高 |
| 个股弱+板块弱 | 看空 | 止损，规避 |

#### 场景一：个股弱但板块强
- **特征**：个股下跌/跌停，但板块内其他个股上涨/涨停
- **常见原因**：停牌复牌补跌、个股利空、前期涨幅大回调
- **结论倾向**：**不宜一味看空**，可能是低吸机会

#### 场景二：个股强但板块弱
- **特征**：个股上涨/涨停，但板块整体走弱
- **常见原因**：游资独立运作、个股独立利好、补涨行情
- **结论倾向**：**谨慎乐观**，持续性存疑

#### 场景三：个股弱且板块弱
- **特征**：个股下跌，板块整体也在下跌
- **结论倾向**：**看空**

#### 场景四：个股强且板块强
- **特征**：个股上涨/涨停，板块整体上涨
- **结论倾向**：**看多**

### 板块强度评估框架

```
检查清单：
├── 1. 板块是否领涨/领跌？
│   ├── 领涨 → 板块强，个股可多持有
│   └── 领跌 → 板块弱，个股冲高卖出
│
├── 2. 板块内涨停/跌停家数？
│   ├── 涨停>3只 → 板块热度高
│   ├── 涨停1-2只 → 板块温和
│   └── 无涨停 → 板块降温
│
├── 3. 板块龙头表现？
│   ├── 龙头继续强势 → 板块主升未结束
│   ├── 龙头高开低走 → 板块可能见顶
│   └── 龙头跌停 → 板块退潮信号
│
└── 4. 板块整体量能？
    ├── 放量 → 资金关注
    └── 缩量 → 资金撤离
```

### 主力吸筹特征（看多信号）

| 分时特征 | 识别方法 | 含义 |
|----------|----------|------|
| 早盘急跌后快速拉回 | 9:30-10:00大幅低开或急跌，10:30前收回大部分 | 量化砸盘+主力承接 |
| 下影线放量 | 下探时成交量骤增，价格快速回升 | 有资金在低位接盘 |
| 尾盘偷偷拉升 | 14:30后缓慢上涨，量能温和放大 | 主力悄悄建仓 |
| 脉冲式放量 | 盘中突然放量拉升2-3%，然后缩量横盘 | 试盘或吸筹 |
| 抗跌性强 | 大盘跳水时跌幅明显小于大盘/板块 | 有资金护盘 |

| 分时特征 | 识别方法 | 含义 |

| 量化特征 | 表现 | 应对 |

## 趋势图生成

```python
import matplotlib.pyplot as plt
import os

plt.rcParams['font.sans-serif'] = ['SimHei', 'DejaVu Sans']
plt.rcParams['axes.unicode_minus'] = False

def generate_trend_chart(dates, closes, highs, lows, stock_code, stock_name, save_path):
    fig, ax = plt.subplots(figsize=(10, 6))
    x = range(len(dates))

    ax.plot(x, closes, 'b-o', linewidth=2, markersize=8, label='收盘价')
    ax.fill_between(x, lows, highs, alpha=0.2, color='gray', label='波动区间')

    for i, (xi, yi) in enumerate(zip(x, closes)):
        ax.annotate(f'{yi:.2f}', xy=(xi, yi), xytext=(0, 10), textcoords='offset points', ha='center')

    ax.set_title(f'{stock_name}({stock_code}) 近7日走势', fontsize=14, fontweight='bold')
    ax.set_xticks(x)
    ax.set_xticklabels([d[5:] for d in dates])
    ax.grid(True, linestyle='--', alpha=0.7)
    ax.legend()

    os.makedirs(os.path.dirname(save_path), exist_ok=True)
    plt.savefig(save_path, dpi=150, bbox_inches='tight')
    plt.close()
```

## Common Mistakes to Avoid

1. **搜索替代API**：严禁用网页搜索结果替代AKShare API获取的精确数据
2. **日期错误**：15:00后必须用当天数据，周末用上一个交易日，开盘前用前一交易日
3. **涨停误判**：收盘价=最高价 ≠ 涨停，必须收盘价 = 涨停价才算涨停
4. **突破误判**：12.27元不可能突破12.41元压力位，必须精确数值比较
5. **旧数据出报告**：API返回数据日期不匹配时，直接用旧数据出报告
6. **放弃获取数据**：API失败后不尝试备用方案直接放弃
7. **热点板块主观化**：热点板块应由涨停数据客观统计，不能因用户持仓而主观调整
8. **孤立看分时**：分时分析必须结合中期趋势和板块联动，不能孤立看待
9. **忽略趋势优先**：趋势未破坏时今日下跌不能轻易看空，趋势已破坏时今日上涨要保持谨慎
10. **盘中数据当收盘**：盘中触及涨停/压力位 ≠ 收盘突破，必须以收盘价为准
