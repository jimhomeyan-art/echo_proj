- 骨架搭建与层级管理

- 动画事件设置
- 导出优化

### 动画状态机
- 设计合理的状态转换逻辑
- 处理动画混合与过渡
- 设置动画事件触发点
- 优化动画性能

### 动画十二法则应用
1. **挤压与拉伸**：体现物体的弹性和质量感
2. **预备动作**：为主要动作做铺垫
3. **舞台调度**：引导观众注意力
4. **连续与关键帧**：选择合适的动画方式
5. **跟随与重叠**：身体部位的延迟运动
6. **缓入缓出**：自然的加速减速
7. **弧线运动**：避免机械的直线运动
8. **次要动作**：丰富主要动作的表现
9. **节奏**：控制动画的时间感
10. **夸张**：适度强调动作特征
11. **扎实绘图**：保持造型稳定
12. **吸引力**：赋予角色魅力

### 动画性能优化
- 控制骨骼数量，避免过度复杂
- 合理设置采样率
- 优化网格顶点数
- 使用动画压缩

## 美术风格参考指南

### 写实风格 (Realistic)

#### 特点
- 追求真实世界的视觉还原
- 高精度贴图和复杂光影
- 注重材质物理属性的准确表现
- 使用PBR工作流

#### 适用类型
- 3A级大作、军事游戏、竞速游戏、体育游戏

#### 技术要点
- 高分辨率贴图（4K）
- 完整PBR材质通道
- 次表面散射、各向异性等高级材质
- 基于物理的光照系统

#### 代表作品
- 《最后生还者》《战神》《荒野大镖客》

---

### 风格化写实 (Stylized Realistic)

#### 特点
- 基于现实但带有艺术化处理
- 夸张的比例和色彩
- 保持可读性的同时追求美感
- 比纯写实更低的制作成本

#### 适用类型
- 动作冒险、角色扮演、开放世界

#### 技术要点
- 适度简化的模型结构
- 饱和度稍高的色彩方案
- 轮廓线可选使用
- 混合使用PBR和非PBR材质

#### 代表作品
- 《原神》《塞尔达传说：旷野之息》《堡垒之夜》

---

### 卡通风格 (Cartoon/Toon)

#### 特点
- 明显的轮廓线
- 简洁的色块填充
- 夸张的形态比例
- 鲜艳明快的配色

#### 适用类型
- 休闲游戏、儿童游戏、横版游戏

#### 技术要点
- Cel Shading着色

## 色彩理论指南

### 色彩三要素
- **色相 (Hue)**：色彩的基本类型（红、蓝、绿等）
- **饱和度 (Saturation)**：色彩的纯度/鲜艳程度
- **明度 (Value/Brightness)**：色彩的明暗程度

### 色轮关系
```
        红
    红橙    红紫
橙              紫
    黄橙    蓝紫
        黄
    黄绿    蓝
        绿
```

### 配色方案

#### 1. 单色配色 (Monochromatic)
- 使用同一色相的不同明度和饱和度变化
- **优点**：和谐统一，不会出错
- **缺点**：可能缺乏对比和活力
- **适用**：UI设计、简约风格游戏

#### 2. 补色配色 (Complementary)
- 使用色轮上相对的两种颜色
- 例如：红-绿、蓝-橙、黄-紫
- **优点**：对比强烈，视觉冲击力强
- **缺点**：需要控制比例，否则刺眼
- **适用**：需要强调对比的场景

#### 3. 分裂补色 (Split-Complementary)
- 一个主色+补色两侧的颜色
- 例如：蓝 + 黄橙 + 红橙
- **优点**：保持对比的同时更加和谐
- **适用**：大多数游戏设计

#### 4. 类似色配色 (Analogous)
- 使用色轮上相邻的3-5种颜色
- 例如：黄-黄绿-绿-蓝绿
- **优点**：自然和谐，过渡流畅
- **适用**：自然场景、环境设计

#### 5. 三角配色 (Triadic)
- 使用色轮上等距的三种颜色
- 例如：红-黄-蓝
- **优点**：丰富多彩，视觉平衡
- **缺点**：需要仔细控制比例
- **适用**：活泼明快的游戏

### 游戏中的功能性色彩
| 用途 | 推荐颜色 | 说明 |
|------|----------|------|
| 生命值 | 红色/绿色 | 约定俗成，玩家直觉理解 |
| 魔法值 | 蓝色 | 传统游戏惯例 |
| 能量/耐力 | 黄色/橙色 | 表示活力 |
| 毒/负面状态 | 紫色/绿色 | 危险但不致命感 |
| 正面增益 | 金色/蓝色 | 积极、提升感 |
| 稀有/传说 | 橙色/紫色 | 特殊、珍贵感 |
| 可交互物 | 高亮黄/白边 | 引导注意力 |

### 阵营色彩
| 阵营 | 常用色彩 | 情感联想 |
|------|----------|----------|
| 正义/光明 | 白、金、蓝 | 纯洁、希望、正直 |
| 邪恶/黑暗 | 黑、紫、红 | 危险、邪恶、力量 |
| 自然/生命 | 绿、棕、蓝 | 生机、和平、治愈 |
| 机械/科技 | 蓝、灰、白 | 冷静、精准、未来 |
| 火焰/破坏 | 红、橙、黄 | 激情、愤怒、毁灭 |
| 冰霜/寒冷 | 白、淡蓝、银 | 寒冷、平静、脆弱 |

### 稀有度配色（通用）
```
普通：灰色/白色
优秀：绿色
稀有：蓝色
史诗：紫色
传说：橙色
神话：红色/金色
```

### 环境氛围色彩

#### 时间段色调
- **清晨**：淡蓝+粉色+金色高光
- **正午**：明亮暖黄+强烈阴影
- **黄昏**：橙红+紫色天空
- **夜晚**：深蓝+月光银白
- **雨天**：灰蓝+低饱和度

#### 场景类型
- **森林**：绿色系为主，棕色点缀
- **沙漠**：黄色/棕色，蓝天对比
- **雪山**：白色/淡蓝，高明度
- **地下城**：暖色火光+冷色石墙
- **未来城市**：霓虹+深色背景

### 实用配色技巧

#### 60-30-10 法则
- **60%**：主色调（背景、大面积）
- **30%**：辅助色（支持元素）
- **10%**：强调色（重点、CTA）

#### 提高可读性
1. 确保文字与背景有足够对比度
2. 避免在复杂背景上放置文字
3. 使用描边或投影增加辨识度
4. 测试色盲友好性

#### 避免的错误
- 同时使用太多饱和色
- 红绿直接相邻（色盲问题）
- 低对比度的功能性UI
- 忽视色彩的文化差异

- **Coolors**：快速生成配色方案

## 美术规范指南

### 资源命名规范

#### 通用规则
- 使用英文命名，禁止中文和特殊字符
- 使用下划线 `_` 分隔单词
- 采用驼峰命名或全小写
- 名称应具有描述性和可读性

#### 2D资源命名
```
[类型]_[功能/角色]_[状态/变体]_[序号]

示例：
UI_btn_confirm_normal.png      # 确认按钮正常状态
UI_btn_confirm_pressed.png     # 确认按钮按下状态
UI_icon_skill_fire_01.png      # 火系技能图标1
CH_hero_warrior_idle_01.png    # 战士英雄待机帧1
BG_forest_layer_01.png         # 森林背景第一层
```

#### 3D资源命名
```
[类型前缀]_[名称]_[部件]_[LOD级别]

前缀：
- CH: Character（角色）
- WP: Weapon（武器）
- ENV: Environment（环境）
- PR: Props（道具）
- VH: Vehicle（载具）
- VFX: Visual Effects（特效）

示例：
CH_Knight_Body_LOD0.fbx
WP_Sword_Fire_LOD0.fbx
ENV_House_Medieval_01_LOD0.fbx
```

#### 贴图命名
```
[模型名]_[贴图类型]

贴图类型缩写：
- _D / _Albedo: 漫反射/基础色
- _N / _Normal: 法线贴图
- _R / _Roughness: 粗糙度
- _M / _Metallic: 金属度
- _AO: 环境光遮蔽
- _E / _Emission: 自发光
- _H / _Height: 高度图
- _Mask: 遮罩贴图

示例：
CH_Knight_Body_Albedo.png
CH_Knight_Body_Normal.png
CH_Knight_Body_Roughness.png
```

### 性能优化指南

#### 贴图优化
- 使用压缩格式（ASTC for mobile, BC7 for PC）
- 开启Mipmap减少远处锯齿
- 合理使用图集减少DrawCall
- 考虑使用通道打包（RGBA存储多张贴图）

#### 模型优化
- 控制面数在预算内
- 移除不可见面
- 合并相同材质的物体
- 使用LOD系统

#### 动画优化
- 减少不必要的关键帧
- 使用动画压缩
- 共享通用动画片段
- 控制骨骼数量
